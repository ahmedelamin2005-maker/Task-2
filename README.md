# Frontend Mentor Order Summary Component Challenge

## Section 1: The Challenge
This is my work for the challenge. The goal is to build a summary card using HTML and CSS. The main focus is to learn the relationship between Parent and Child elements. It also focuses on using CSS Inheritance well. 



## Section 2: My Process

### Built with
* Semantic HTML5
* CSS Custom Properties
* Flexbox

### What I learned
Learning about Parent and Child parts helped me write much less code. CSS Inheritance made things very easy. I set the font and color once in the main box. All the smaller parts inside copied these styles automatically. I saved time and did not repeat the font rules.

## Section 3: Key Learnings
I read about CSS Specificity. This means knowing which style wins when a parent and child have different rules. I also read about Explicit and Implicit Inheritance. This explains why some styles copy automatically, like text color, while others do not, like borders.

I used the inherit word as a neat trick. I used it on buttons and links so they match the page without writing extra code. I also used Flexbox on the parent box to easily line up the icon, price, and link inside.
